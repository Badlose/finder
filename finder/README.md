## 1. Установка

  Клонируйте репозиторий:
    ```
    bash
    git clone <https://github.com/Badlose/finder>
    cd finder
    ```

## 2. Сборка

  Скомпилируйте проект с помощью Maven: Откройте терминал или командную строку, перейдите в корневую директорию 
  проекта (finder/) и выполните команду:

```
  bash
  mvn clean install
```

## 3. Создание тестового .xlsx файла

Создайте новый файл Excel.

В первом столбце (A) введите целые числа, каждое в своей строке. Например:

|   | A  | 
|---|----|
| 1 | 15 | 
| 2 | 7  | 
| 3 | 23 | 
| 4 | 5  | 
| 5 | 10 | 
| 6 | 1  | 
| 7 | 18 | 
| 8 | 8  | 


Сохраните этот файл как numbers.xlsx (или любое другое имя) в удобном для вас месте на вашем компьютере. 
Запомните полный путь к этому файлу.


## 4. Запуск приложения
После успешной сборки JAR-файл будет находиться в директории target/ вашего проекта.

Перейдите в директорию target/:

```
bash

cd target
```
Запустите JAR-файл:

```
bash

java -jar finder-0.0.1-SNAPSHOT.jar
```
(Имя JAR-файла может немного отличаться в зависимости от версии Spring Boot и имени вашего артефакта).

Приложение запустится, и вы увидите логи в консоли, указывающие на то, что Spring Boot стартовал.

## 5. Доступ к Swagger UI
   После запуска приложения вы сможете получить доступ к Swagger UI, открыв в браузере следующий URL:

http://localhost:8080/swagger-ui/index.html


## 6. Использование API

   В Swagger UI вы увидите единственный эндпоинт /api/find.
   Нажмите на него, затем нажмите кнопку “Try it out”.

   В поле filePath введите полный абсолютный путь к вашему файлу numbers.xlsx, который вы создали на шаге 3. 
   Важно: Убедитесь, что путь указан корректно для операционной системы, на которой вы запускаете приложение 
   (например, C:/Users/YourUser/Documents/numbers.xlsx для Windows или 
   /home/youruser/documents/numbers.xlsx для Linux/macOS).
   
   В поле n введите число, определяющее, какое минимальное число вы хотите найти. 
   Например:
   1 для самого минимального числа.
   3 для третьего минимального числа.
   Нажмите кнопку “Execute”.
   Вы увидите ответ:

Если все успешно, в разделе “Response body” будет указано N-ное минимальное число.

В случае ошибок (неверный путь, некорректное N, проблемы с файлом) будет соответствующее сообщение об ошибке.
